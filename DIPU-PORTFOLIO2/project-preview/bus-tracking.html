About This Project

Students never know where the college bus is. Is it five minutes away or twenty? Has it already left? This application removes the guesswork by showing live bus positions on a map, updated in real time.

No page refresh. No delay. Just a moving dot on a map that tells you exactly where your bus is right now.

Problems Solved

Building a real-time tracking system requires handling several technical challenges. Here's what this project addresses.

Real-Time Data Streaming

Problem: Traditional HTTP requires constant polling to check for updates. This wastes bandwidth and introduces delays.

Solution: Socket.io establishes persistent WebSocket connections. The server pushes location updates the moment they arrive — no polling, no delay.

Live Map Updates

Problem: Static maps require page refresh to show new positions. Users lose context and the experience feels broken.

Solution: Leaflet.js markers update their coordinates dynamically when Socket.io events arrive. The map stays interactive while markers move smoothly.

Driver Authentication

Problem: Anyone could send fake location data and corrupt the tracking system. Driver identities need verification.

Solution: Passport.js handles authentication. Passwords are hashed with bcrypt. Only verified drivers can broadcast their location.

Session Persistence

Problem: In-memory sessions are lost when the server restarts. Drivers would need to log in again constantly.

Solution: Connect-Mongo stores sessions in MongoDB. Sessions survive server restarts and can be shared across multiple server instances.

Multiple Bus Tracking

Problem: A campus has multiple buses on different routes. Each needs to be tracked independently on the same map.

Solution: Each bus has a unique identifier. The map maintains separate markers for each bus, all updating simultaneously.

Cross-Device Access

Problem: Students use phones, tablets, and laptops. The interface needs to work everywhere without separate apps.

Solution: Responsive design with Tailwind CSS. The same web app works on all screen sizes. No app store downloads needed.

Technology Stack

Each tool was chosen for a specific purpose. Here's the complete stack and what each piece handles.

Node.js

Server runtime. Handles all backend logic, processes location data, and manages WebSocket connections.

Express.js

Web framework. Defines routes, handles middleware, manages sessions, and structures the application.

MongoDB

Database. Stores driver accounts, bus information, and persistent sessions via Connect-Mongo.

Socket.io

WebSocket library. Streams location data from drivers to all connected student clients in real time.

Leaflet.js

Mapping library. Renders interactive maps with OpenStreetMap tiles and handles live marker positioning.

Passport.js

Authentication middleware. Handles driver login with local strategy. Integrates with Express sessions.

EJS Templates

Server-side rendering. Generates HTML on the server with dynamic data before sending to browser.

Tailwind CSS

Utility-first CSS framework. Provides responsive design utilities for cross-device compatibility.
